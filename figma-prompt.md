# Figma Make Prompt

## 프로젝트 컨텍스트

- 프로젝트: JIRA Release Note 기반 비개발자용 업데이트 공지 자동 생성 서비스
- 기존 화면: 없음 (신규 프로젝트)
- 기술 스택: 미정 (기획 단계)
- 대상 사용자: 물류운영팀, 영업팀, 고객만족팀 등 비개발 직군
- 디바이스: 데스크톱 웹 브라우저

## 디자인 방향

- 비개발자가 직관적으로 사용할 수 있는 깔끔하고 친근한 UI
- 업무 도구 특성에 맞는 전문적이면서도 부담 없는 톤
- 카드 기반 레이아웃으로 공지 항목을 시각적으로 구분
- 기존 화면이 없으므로, 모던 SaaS 대시보드 스타일을 참고

## 생성할 화면

### 화면 1: 로그인 페이지

- 목적: 사용자 인증 (이메일/비밀번호 로그인)
- 주요 구성요소: 이메일 입력, 비밀번호 입력, 로그인 버튼, 회원가입 링크, 로그인 실패 시 오류 메시지 영역
- 레이아웃: 중앙 정렬 카드 폼
- 인터랙션: 입력 필드 포커스, 로그인 버튼 클릭, 오류 상태 표시
- 데이터 예시: 이메일 "user@company.com", 오류 메시지 "이메일 또는 비밀번호가 올바르지 않습니다"

### 화면 2: 회원가입 페이지

- 목적: 신규 사용자 계정 생성
- 주요 구성요소: 이메일 입력, 비밀번호 입력, 비밀번호 확인, 가입 버튼, 로그인 페이지 링크
- 레이아웃: 중앙 정렬 카드 폼 (로그인과 일관된 스타일)
- 인터랙션: 입력 필드 유효성 검증, 가입 완료 후 로그인 페이지 이동
- 데이터 예시: 이메일 "newuser@company.com"

### 화면 3: 대시보드

- 목적: 서비스의 메인 허브. 공지 생성 진입점 + 발송 이력 조회
- 주요 구성요소:
  - 상단 헤더 (서비스명, 사용자 정보, 로그아웃 버튼)
  - "새 공지 생성" CTA 버튼 (강조)
  - 발송 이력 테이블 (발송일시, 제목, 수신자 수, 상태)
  - 이력 항목 클릭 시 상세 보기 이동
- 레이아웃: 상단 헤더 + 메인 콘텐츠 영역 (CTA + 테이블)
- 인터랙션: CTA 클릭 → URL 입력 페이지, 이력 행 클릭 → 상세 페이지
- 데이터 예시: 발송 이력 3-5건 (제목: "2026년 3월 시스템 업데이트 안내", 상태: 발송 성공/실패)

### 화면 4: URL 입력 페이지

- 목적: JIRA Release Note URL을 입력하여 공지 초안 생성을 요청
- 주요 구성요소:
  - 페이지 제목 및 안내 텍스트
  - URL 입력 필드 (기본 2개)
  - 각 필드 옆 삭제 버튼 (기본 필드 제외)
  - "URL 추가" 버튼
  - URL 형식 유효성 오류 메시지 영역
  - "초안 생성" 버튼 (Primary)
  - 로딩 상태 (초안 생성 중)
  - JIRA 연동 오류 메시지 영역
- 레이아웃: 단일 컬럼, 카드 내 폼
- 인터랙션: URL 추가/삭제, 유효성 검증 피드백, 생성 버튼 클릭 → 로딩 → 편집 페이지 이동
- 데이터 예시: URL "https://jira.company.com/projects/PROJ/versions/12345", 오류 "유효한 JIRA Release Note URL을 입력해주세요"

### 화면 5: 초안 편집 페이지

- 목적: 자동 생성된 공지 항목을 검토하고 편집
- 주요 구성요소:
  - 공지 항목 카드 목록 (드래그 앤 드롭 가능)
  - 각 카드: 제목, 설명 텍스트, 카테고리 태그, Slack 링크, 스크린샷 썸네일 영역
  - 카드별 액션: 편집, 삭제, 스크린샷 첨부 버튼
  - 다중 선택 체크박스 + "선택 항목 병합" 버튼
  - 카테고리 선택 드롭다운 (기능 개선, 버그 수정, 신규 기능, UI/UX 변경 등)
  - 스크린샷 첨부 영역 (항목당 최대 5개, 썸네일 표시)
  - AI 변환 실패 항목 표시 (원본 JIRA 텍스트 + "수동 편집 필요" 배지)
  - 하단 액션: "URL 재입력" 버튼 (Secondary), "이메일 발송" 버튼 (Primary)
- 레이아웃: 단일 컬럼, 카드 리스트
- 인터랙션: 드래그 앤 드롭 순서 변경, 인라인 편집, 항목 선택 후 병합, 스크린샷 업로드, 카테고리 드롭다운
- 데이터 예시:
  - 항목 1: "[기능 개선] 주문 조회 화면 검색 필터 추가" / "이제 주문 상태, 날짜 범위별로 필터링하여 원하는 주문을 빠르게 찾을 수 있습니다."
  - 항목 2: "[버그 수정] 배송 현황 표시 오류 해결" / "일부 주문에서 배송 현황이 실시간으로 반영되지 않던 문제가 해결되었습니다." + Slack 링크
  - 항목 3 (AI 실패): 원본 JIRA 텍스트 표시 + "수동 편집 필요" 배지

### 화면 6: 이메일 설정 및 미리보기 페이지

- 목적: 이메일 발송 설정 및 최종 HTML 미리보기 확인 후 발송
- 주요 구성요소:
  - 설정 영역: 이메일 제목 입력, 발신자 이름 입력, 수신자 이메일 입력 (자동 완성, 다중 입력)
  - 수신자 태그 형태 표시 (추가/삭제 가능)
  - HTML 미리보기 영역 (카테고리별 구분, 이미지 인라인 표시)
  - 발송 버튼, 발송 성공/실패 메시지 영역
- 레이아웃: 좌측 설정 패널 + 우측 미리보기 (또는 상/하 분할)
- 인터랙션: 수신자 자동 완성, 태그 추가/삭제, 미리보기 실시간 반영, 발송 클릭
- 데이터 예시: 제목 "2026년 3월 시스템 업데이트 안내", 발신자 "IT기획팀", 수신자 ["logistics@company.com", "sales@company.com"]

### 화면 7: 발송 이력 상세 페이지

- 목적: 이전에 발송한 이메일의 상세 내용 확인 및 재발송
- 주요 구성요소:
  - 발송 정보: 발송일시, 제목, 발신자 이름, 수신자 목록, 발송 상태
  - 발송된 공지 내용 (HTML 렌더링)
  - "재발송" 버튼 → 이메일 설정 페이지로 이동 (기존 내용 프리필)
  - "목록으로" 버튼 → 대시보드 복귀
- 레이아웃: 단일 컬럼, 정보 카드 + 공지 내용 미리보기
- 인터랙션: 재발송 클릭, 목록 복귀
- 데이터 예시: "2026년 3월 시스템 업데이트 안내" 발송 상세, 수신자 3명, 발송 성공

## 기존 화면과의 연관

- 로그인 → 대시보드 → URL 입력 → 초안 편집 → 이메일 설정/미리보기 → 발송 완료 → 대시보드
- 대시보드에서 발송 이력 선택 → 이력 상세 → 재발송 시 이메일 설정 페이지
- 모든 화면 상단 헤더는 공통 컴포넌트 (서비스명, 사용자 정보, 로그아웃)

## 참고

- 7개 화면 전체가 일관된 디자인 시스템으로 제작되어야 합니다
- 비개발자 대상 서비스이므로 직관적이고 친근한 UI를 지향합니다
- 카드/테이블 기반 레이아웃으로 정보를 구조화하세요
- 초안 편집 페이지가 가장 복잡하므로, 드래그 앤 드롭, 다중 선택, 병합 등의 인터랙션을 시각적으로 명확히 표현해주세요
